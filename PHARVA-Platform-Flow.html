01 — Overview

What is PHARVA?

PHARVA is a full-stack healthcare platform built for the Nigerian pharmaceutical supply chain. It connects pharmacies, healthcare facilities, wholesalers, distributors, and patients on a single verified network — enabling them to redistribute near-expiry medications, manage inventory intelligently, trade securely, respond to emergency drug requests, and manage patient care.

The platform has three entry points: a mobile app (React Native / Expo) for on-the-go use, a web dashboard (Next.js) for full desktop operations, and a REST API + real-time Socket.io backend powering both.

Core problem PHARVA solves: Nigerian pharmacies lose an estimated ₦200B+ annually to medication expiry waste. PHARVA creates a trusted marketplace for redistributing near-expiry stock between verified facilities — turning waste into recovered revenue.

03 — Onboarding

Registration & Account Setup

Step-by-Step Registration (Medical / Non-Medical)

  1. Personal details — Full name, email address, password (min 8 characters), and role selection (MEDICAL / NON_MEDICAL / PATIENT).

  2. Facility details — Pharmacy or business name, facility type (Community Pharmacy / Hospital Pharmacy / Pharmacy Chain / Healthcare Facility / Distributor / Wholesaler), physical address, city, and state.

  3. License / registration — PCN license number or professional registration number. Optional at signup — can be added later in Settings. Without it, the account remains at BASIC verification level.

  4. Account created — A Wallet is automatically created with ₦0 balance. The user receives JWT access and refresh tokens and is logged in immediately.

Verification Levels

Level Status What it unlocks
BASIC No license submitted Browse marketplace, limited trading
PENDING License submitted, awaiting review Full platform access (trading, Emergency Rx)
VERIFIED License approved by PHARVA team Full access + trust badge on listings

Patient registration is simpler — no facility details or license number required. Patients are typically linked to a pharmacy by their pharmacist who manages their care profile on the platform.

04 — Feature Flow

Inventory Management

Every pharmacy's core activity. This is where drugs are logged, tracked, and monitored for expiry.

Adding a Drug to Inventory

  1. Drug details — Generic name, brand name, dosage form (tablet / syrup / injection / capsule / cream), strength, manufacturer.

  2. Stock details — Quantity in stock, unit (tablets / bottles / vials), batch number, cost price per unit, selling price per unit.

  3. Expiry date — The most critical field. PHARVA uses this to monitor and trigger all expiry-related alerts.

  4. Storage conditions — Refrigerated, room temperature, controlled substance flag.

  5. Saved — Item appears in inventory dashboard with real-time status indicators.

Dead Stock Detection

Items that have been in inventory without movement for an extended period are automatically flagged as DEAD STOCK. These appear in a dedicated dashboard section with a prompt to list them on the marketplace or investigate demand.

Inventory Status Labels

  • IN STOCK — Sufficient quantity, healthy expiry window
  • EXPIRING SOON — Within 90 days
  • CRITICAL — Within 30 days
  • DEAD STOCK — No movement detected
  • OUT OF STOCK — Quantity at zero
05 — Feature Flow

Marketplace — Buying & Selling

The marketplace is the heart of PHARVA's redistribution mission. Pharmacies list surplus or near-expiry stock; other verified facilities purchase it at a discounted price before it becomes waste.

Creating a Listing (Seller Flow)

  1. Select inventory item — Choose from existing inventory or create a new entry.

  2. Set listing details — Listing price (typically below cost to move quickly), minimum order quantity, description, and condition notes.

  3. Set visibility — Public (all verified users) or restricted to specific facilities / network.

  4. Publish — Listing goes live. Status: ACTIVE. Appears in search results for nearby facilities.

  5. Receive orders — Buyers place orders. Seller gets a real-time notification.

Placing an Order (Buyer Flow)

  1. Browse / search — Filter by drug name, dosage form, location, expiry date range, price. Verified sellers are highlighted.

  2. View listing — Full details: drug info, batch number, expiry date, seller name and verification status, price, quantity available.

  3. Place order — Select quantity, confirm. Order is created and buyer's wallet is immediately debited.

  4. Funds go into escrow — Money is NOT sent to seller yet. It is held securely in escrow pending delivery.

  5. Track order — Real-time status updates as the seller processes and ships.

Cancellation protection: If an order is cancelled while still in PENDING status (before seller confirms), the full ₦50,000 is refunded from escrow back to the buyer's wallet balance. No loss to either party.

06 — Feature Flow

Emergency Rx — Urgent Drug Requests

When a patient urgently needs a medication that a pharmacy does not have in stock, the pharmacist can broadcast an Emergency Rx request to all nearby verified pharmacies on the PHARVA network. This is a real-time, location-aware alert system.

Sending an Emergency Rx Request (Requesting Pharmacy)

  1. Open Emergency Rx — Tap the LIVE EmergencyRx feature in the navigation.

  2. Enter drug details — Drug name, dosage form, strength, quantity needed, patient age group, urgency level.

  3. Add patient context — Brief clinical note (optional but helpful) — e.g., "Post-surgical patient, needs IV Cefuroxime 750mg."

  4. Broadcast — Request is sent in real-time via Socket.io to all verified pharmacies within the configured radius.

  5. Receive responses — Pharmacies that have the drug respond with their availability and price. The requesting pharmacist sees all responses in real time.

  6. Coordinate — Arrange for patient to go to the responding pharmacy, or arrange transfer of drugs.

Responding to an Emergency Rx (Responding Pharmacy)

  1. Receive real-time alert — A push notification and in-app alert appears: "URGENT: Cefuroxime 750mg IV needed nearby."

  2. Check stock — Quickly verify availability in their own inventory.

  3. Respond — Confirm availability, state quantity and price. Response is sent immediately to the requesting pharmacy.

  4. Serve patient — Patient is directed to them, or the drug is transferred. Transaction can proceed through the marketplace if needed for payment tracking.

Why this matters: In Nigerian hospitals, patients frequently cannot get drugs from the hospital pharmacy and must send family members to multiple community pharmacies. Emergency Rx eliminates this by creating a 60-second broadcast that reaches dozens of pharmacies simultaneously.

07 — Feature Flow

Patient Management & Medication Adherence

Medical professionals can register and manage their patients on PHARVA, set up medication schedules, and track how well patients are adhering to their treatment plans.

Registering a Patient (Pharmacist's View)

  1. Add new patient — Name, date of birth, gender, phone number, email (optional), blood group, genotype, known allergies.

  2. Medical history — Chronic conditions, current medications, past surgeries — for context when handling their prescriptions.

  3. Patient is linked — The patient is now associated with this pharmacy/facility. The pharmacist can manage all their care from here.

Setting Up a Medication Schedule

  1. Select patient — Open the patient's profile from the Patients list.

  2. Create schedule — Drug name, dosage (e.g., "500mg"), frequency (e.g., twice daily), route (oral / injection / topical), start date, end date, special instructions (e.g., "take with food").

  3. Schedule is live — The patient sees their medication schedule in their PHARVA app. They receive reminders at the configured times.

Adherence Tracking

For each scheduled dose, the patient logs one of three statuses:

  • TAKEN — Dose was taken on time
  • MISSED — Dose was not taken
  • SKIPPED — Intentionally skipped (e.g., side effects)

The pharmacist can view the patient's adherence history and follow up on persistent missed doses. This is especially valuable for chronic disease management (hypertension, diabetes, TB, HIV).

08 — Feature Flow

Wallet & Payments

Every MEDICAL and NON_MEDICAL account has a PHARVA Wallet — a stored-value account that powers all transactions on the platform. Payments are handled via Paystack, Nigeria's leading payment gateway.

Wallet Structure

Component What it holds
Balance Funds available for spending on the marketplace or withdrawal to your bank account
Escrow Funds locked in active orders. Cannot be spent or withdrawn until order is completed or cancelled

Funding Your Wallet

  1. Navigate to Wallet and tap "Fund Wallet."

  2. Enter the amount to add.

  3. Complete payment via Paystack (card, bank transfer, USSD).

  4. Wallet balance updated immediately on successful payment confirmation from Paystack webhook.

Withdrawing from Wallet

  1. Navigate to Wallet and tap "Withdraw."

  2. Enter bank account details (one-time setup) and amount.

  3. Payout initiated via Paystack Transfers API.

  4. Funds credited to bank account within minutes (instant transfer) or up to 24 hours.

Note: Funds held in Escrow cannot be withdrawn. You must wait for the associated order to reach COMPLETED (escrow released to seller) or CANCELLED (refunded to buyer) status first.

Transaction Types

  • CREDIT — Wallet top-up via Paystack, sale proceeds released from escrow
  • DEBIT — Purchase on marketplace, withdrawal to bank
  • ESCROW_HOLD — Funds locked when buyer places an order
  • ESCROW_RELEASE — Funds released to seller on order completion
  • REFUND — Escrow returned to buyer on order cancellation

Type Use Case Example

GROUP Multi-member channel for communities and associations Lagos Island Pharmacists Association, PCN Zone 3 Group

10 — Feature Flow

MediCareer — Pharmacy Job Board

A professional job board built into PHARVA specifically for the pharmaceutical sector. Facilities can post roles and professionals can apply — all within the verified network.

Posting a Job (Facility)

  1. Job details — Title, description, requirements, responsibilities, location, salary range (optional).

  2. Job type — FULL_TIME PART_TIME LOCUM INTERNSHIP CONTRACT.

  3. Application deadline — After this date, no new applications are accepted and the listing is automatically closed.

  4. Published — Visible to all verified professionals on the platform.

Applying for a Job (Professional)

  1. Browse the job board, filter by type, location, or specialty.

  2. View full job details including the posting facility's name and verification status.

  3. Submit application — includes a cover message and links to their PHARVA profile (which shows their verification level, facility, and professional details).

  4. Facility receives notification and reviews applicants through the platform.

Security Model

  • All passwords hashed with bcrypt (10 salt rounds) — never stored in plain text
  • JWT access tokens expire in 15 minutes; refresh tokens in 7 days
  • Refresh tokens are stored in the database and rotated on every use (old token invalidated)
  • Logout invalidates the refresh token in the database immediately
  • All inputs validated with Zod schemas before hitting the database
  • File uploads are processed and hosted via Cloudinary — no raw file storage on the server
  • Paystack webhooks verified with signature to prevent fraudulent payment confirmations
